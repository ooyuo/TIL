여러개의 input 상태 관리하기
=============  
  
  
  
input의 개수가 여러개 일때 가장 좋은 방법은 input에 name을 설정하고  
이벤트가 발생했을 때 이 값을 참조하는 것이다.  
그리고, useState에서 문자열이 아니라 객체 형태의 상태를 관리해주어야 한다.  

리액트 상태에서 객체를 수정해야 할 때  
<pre><code>inputs[name] = value;</code></pre>  
이런 식으로 직접 수정하면 안된다.  

이 방법을 권장한다.  
새로운 객체를 만들어서 새로운 객체에 변화를 주고, 이를 상태로 사용해 주어야한다.  

<pre><code>setInputs({
    ...inputs,
    [name]: value
});</code></pre>  

## 참고  
  
- ... 는 spread문법이다. 객체의 내용을 모두 "펼쳐서"기존 객체를 복사해준다.  
  이러한 작업을 '불변성을 지킨다'라고 한다.  
- 리액트에서는 불변성을 지켜주어야만 컴포넌트 업데이트 성능 최적화를 제대로 할 수 있다.  


## 핵심  
  
리액트에서 객체를 업데이트하게 될 때에는 기존 객체를 직접 수정하지 말고,  
__새로운 객체를 만들어 객체로 상태를 관리해주자.__
